# User Profile — Rohan

## Communication

- Explain things in **simple English**. If a concept is technical, give a plain-language analogy first.
- **Roman Urdu is welcome** when Rohan uses it — respond in whichever language he uses in that message.
- Keep explanations short. If the answer fits in two sentences, do not write a paragraph.

## Coding Preferences

- **Keep code simple.** If there is a straightforward way and a clever way, always pick the straightforward way.
- **No overengineering.** Do not add state management libraries, extra abstractions, or "future-proof" patterns unless explicitly asked.
- **Ask before deleting any file.** Even if a file looks unused, confirm before removing it.
- **Mobile first, always.** Every UI change must look good on a phone screen before anything else.

## Priority Features for This Site

When suggesting improvements or building new features, prioritise in this order:

1. **Mobile responsiveness** — site must work perfectly at 375 px width
2. **Quote calculator** — instant price estimate (service type + size → price range)
3. **Booking / contact form** — easy for a customer to submit a request
4. **WhatsApp button + floating call button** — always visible, easy to tap on mobile
5. **Lead generation** — capture name, phone, service, message and store or forward them

## Things to Avoid

- Long code explanations when Rohan just wants the working code
- Suggesting rewrites of things that already work
- Adding features that were not asked for
- Verbose trailing summaries after completing a task
